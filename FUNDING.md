# Funding And Credits

`blackhole-threats` is MIT-licensed open source software with first-party source
builds, container images, Debian packages, a signed APT repository, and public
release automation.

Support helps keep that operational surface maintained.

## What Support Funds

Funding, sponsorship, hosted credits, and tooling credits may be used for:

- CI, build, and validation capacity
- release engineering and package publication
- test environments for Debian, container, and routing behavior
- documentation and website maintenance
- dependency and supply-chain maintenance
- AI-assisted triage, documentation, testing, and development, with maintainer
  review before changes are merged

## Current Support Channels

## Credits Or In-Kind Support

For provider-specific open source credit programs, this repository's public
project-health files are intended to provide review context:

- [LICENSE](LICENSE)
- [README.md](README.md)
- [CONTRIBUTING.md](CONTRIBUTING.md)
- [SECURITY.md](SECURITY.md)
- [GOVERNANCE.md](GOVERNANCE.md)
- [ROADMAP.md](ROADMAP.md)

Restricted credits or infrastructure should support the public project and must
not require private control over releases, signing, vulnerability handling, or
the project roadmap.

## Transparency

Support is appreciated, but the project remains open source and maintainer-led.
Features, fixes, and releases continue to be prioritized by operator impact,
safety, maintenance cost, and available maintainer time.
